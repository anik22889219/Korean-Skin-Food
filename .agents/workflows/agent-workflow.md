# AGENT WORKFLOW — KOREAN SKIN FOOD

## STEP 1 — ANALYZE

Before implementation:

- Understand the request
- Analyze affected modules
- Detect dependencies
- Identify risks
- Estimate scalability impact

---

## STEP 2 — PLAN

Create an execution strategy:

- Define architecture
- Identify reusable modules
- Define database impact
- Define API structure
- Define frontend impact

---

## STEP 3 — IMPLEMENT

During implementation:

- Follow project architecture
- Keep code modular
- Maintain naming consistency
- Avoid shortcuts
- Write scalable logic
- Keep business logic isolated

---

## STEP 4 — VALIDATE

After implementation:

- Test functionality
- Test responsiveness
- Test APIs
- Test edge cases
- Check console logs
- Validate performance

---

## STEP 5 — OPTIMIZE

Then optimize:

- Repetitive logic
- Database queries
- Rendering performance
- Caching strategy
- API efficiency

---

## STEP 6 — DOCUMENT

Always provide:

1. What changed
2. Which files changed
3. Why changes were made
4. Potential risks
5. Suggested future improvements

---

# AGENT RESPONSE FORMAT

Every response should include:

1. Problem Analysis
2. Root Cause
3. Execution Plan
4. Implementation Details
5. Risks
6. Optimization Suggestions

---

# EXECUTION STANDARD

Every implementation must be:

- Safe
- Production-ready
- Scalable
- Maintainable
- Secure
- Optimized

The agent must always prioritize long-term system stability over short-term shortcuts.
